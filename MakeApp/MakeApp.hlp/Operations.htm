MakeApp Operations

Specify Base Directory

Brings up a directory dialog box. The user must specify the base directory for the application and the "Library Project".

Name Project

Invokes a dialog box which asks for three items:

  • The Project Name, usually one word
  • The Visible Application Name (which will show up on the windows title bar)
  • The Window Description (which will also show up on the window title bar)
When OK is selected the new application is created and is writen to the new directory. The directory structure looks a little like this:
  • ProjectName
    • ProjectName -- The directory in which the the VS solution file is stored
    • ProjectName.prj -- The directory in which the project files are stored
    • ProjectName.hlp -- The directory in which the help files are stored
  • Library -- The top most directory of the Library

Fix SlickEdit

SlickEdit's order of commands places the most important one near the bottom of the Build Menu. The idea of this command is to reorder the Build Menu. The command opens a file dialog box to load, modifyand display the ".vpj" file. It does not save the file. Use the Save file command to save the file.

List Fonts

Open a dialog box to specify the first letter of the fonts that should be displayed. The font name and an example string using the font are displayed for each font with the beginning letters seleected.

Calibrate Font Size

This invokes a dialog box whose purpose is to find a constant that will make the Courier New font extend to a certain physical distance. The options dialog allows one to select "Landscape" or "Portrait" mode for output and the number of characters printed is dependent on the "Orientation":

  • Portrait Orientation will cause 80 characters to be printed
  • Landscape Orientation will cause 100 characters to be printed

Use the vertical bars to measure the width of the line.

A calibration cycle is composed of the following steps:

  • Choose the Orientation in the Options Dialog
  • Choose the Calibrate on the Tool Bar or Tools Menu
  • Select the Display or Printer to calibrate
  • Choose a new calibration value and insert it into the edit window, or
  • Check one of the radio buttons (round) below the values shown:
    • Left of Left -- New value for calibration value is below left hand value (i.e. lower bound)
    • Left Middle -- New value is half way between left hand value and current calibration value
    • Right Middle -- New value is half way between right hand value and current calibration value
    • Right of Right -- New value is larger than right hand value (i.e. upper bound)
  • Select OK
  • Using a ruler measure the line produced. This may involve printing the content of the screen.

There is no reason to suppose that the dimension on the screen need be perfectly correct. Changing the point size in the code will proportionally change the displayed characters.

The value decide may be copied down and transferred to other programs using their Options dialog and it is saved in the initialization file for that application.

Options

The options dialog allows specifying the Display and Printer Font Size Scale Factor, the printer orientation and the printer margins.

Save File As (Toolbar Floppy Image)

Displays a Save File Dialog and saves the file to the file named. A backup file is created that has a time-stamped extension. Only five backups will be maintained.

About (Toolbar Question Mark)

Displays the About Box. The about box uses text from the resource file (TestApp.rc) for the output information. There is a standard about box dialog in the Library.

Exit (Toolbar X image)

Exit the application with no reservations.
